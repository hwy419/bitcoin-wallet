## PROJECT CONTEXT
================================================================================

### Product Overview
Bitcoin Wallet Chrome Extension - A secure, user-friendly browser extension for managing Bitcoin. Think MetaMask but for Bitcoin. Built with React, TypeScript, Tailwind CSS, and Manifest V3.

### Design Philosophy
- **Security-First**: Users need confidence their Bitcoin is safe
- **Simplicity**: Complex crypto operations hidden behind simple interfaces
- **Transparency**: Clear feedback on all actions and states
- **Trustworthiness**: Professional design that conveys reliability
- **Accessibility**: Usable by all, including those with disabilities

### Extension Constraints
- Fixed popup size: 600x400px
- Chrome extension environment (limited viewport)
- Must work in popup, options page, and potential side panel
- Performance considerations (lightweight, fast loading)
- No external font loading (Chrome extension CSP restrictions)

### Target Users
- Cryptocurrency beginners needing intuitive wallet
- Experienced users wanting MetaMask-like Bitcoin experience
- Developers testing Bitcoin applications on testnet
- Privacy-conscious users preferring browser-based solutions

### Technical Stack
- React 18 + TypeScript
- Tailwind CSS for styling
- Chrome Extension Manifest V3
- No custom fonts (system fonts only)

### Reference
See ARCHITECTURE.md for technical details and user flows.

- Transaction ID display with copy button

**Receive**
- Current receiving address display
- QR code generation and display

**Transaction History**
- Transaction list (chronological)
- Transaction cards (sent/received visual distinction)

**Settings**
- Auto-lock timer

### 3. User Experience Design
Craft intuitive user flows:

**Information Architecture**
- Clear navigation structure
- Logical screen hierarchy
- Minimal clicks to complete tasks
- Consistent layout patterns

**Interaction Design**
- Button states (default, hover, active, disabled, loading)
- Form validation (inline, real-time feedback)
- Error handling and recovery
- Loading states and progress indicators
- Animations and transitions (subtle, purposeful)
- Keyboard navigation support

**Microinteractions**
- Copy to clipboard feedback
- Transaction confirmation animations
- Success/error toast notifications
- Hover states and tooltips
- Focus indicators for accessibility

### 4. Accessibility (A11y)
Ensure WCAG 2.1 AA compliance:

- Color contrast ratios (4.5:1 for text, 3:1 for UI elements)
- Keyboard navigation for all interactions
- Focus indicators clearly visible
- ARIA labels for screen readers
- Alt text for images and icons
- Form labels and error associations
- Skip navigation links where appropriate
- Readable font sizes (minimum 14px for body text)

### 5. Responsive Design
Adapt to different contexts:

- Popup (600x400px default)
- Options page (full browser tab)
- Different browser zoom levels
- High DPI displays (Retina)

### 6. Visual Design
Create beautiful, trustworthy aesthetics:

- Bitcoin-themed color palette
- Modern, clean interface
- Appropriate use of whitespace
- Visual hierarchy (guide user attention)
- Consistent iconography
- Professional polish

### 7. Prototyping & Iteration
- Create interactive prototypes for user testing
- Iterate based on feedback
- A/B test design alternatives
- Gather user insights
- Refine designs continuously

## USER FLOW DESIGNS
================================================================================

### Critical User Flows to Design

1. **First-Time Wallet Creation**
   - Welcome → Create/Import choice → Address type → Password → Seed display → Seed confirm → Success
   - Focus: Security warnings, clear instructions, trust-building

2. **Wallet Unlock**
   - Locked screen → Password entry → Dashboard
   - Focus: Speed, security, clear feedback

3. **Send Bitcoin**
   - Dashboard → Send → Address + Amount → Fee selection → Review → Confirm → Success
   - Focus: Clarity, error prevention, transaction confidence

4. **Receive Bitcoin**
   - Dashboard → Receive → Address + QR display
   - Focus: Easy sharing, copy functionality, clarity

5. **Account Management**
   - Dashboard → Account dropdown → Select account / Create new / Rename
   - Focus: MetaMask-style familiarity, ease of use

### Flow Design Principles

**Progressive Disclosure**
- Show essential information first
- Advanced options available but not prominent
- Reduce cognitive load

**Error Prevention**
- Validate inputs in real-time
- Clear error messages
- Disable actions when requirements not met

**Clear Feedback**
- Loading states for all async operations
- Success confirmations
- Error explanations with recovery actions

## BEST PRACTICES
================================================================================

### Crypto Wallet Design Principles

**Trust & Security**
- Professional, polished aesthetics
- Clear security warnings
- Explicit confirmations for sensitive actions
- Never hide important security information

**Clarity**
- Clear labels and instructions
- Plain language (avoid jargon when possible)
- Explain Bitcoin concepts when needed
- Show what will happen before it happens

**Error Prevention**
- Validate inputs before submission
- Confirm destructive or irreversible actions
- Provide examples for correct input formats
- Disable actions when requirements not met

**Feedback**
- Show loading states for all async operations
- Confirm successful operations
- Explain errors clearly with recovery actions
- Display transaction status prominently
