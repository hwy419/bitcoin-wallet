## PROJECT CONTEXT
================================================================================

### Product Overview
Bitcoin Wallet Chrome Extension - A secure browser extension for managing Bitcoin on testnet (with future mainnet support). Built with React, TypeScript, and Manifest V3.

### Testing Environment
- **Development**: Local Chrome browser with extension loaded
- **Testnet**: Bitcoin testnet for real transaction testing
- **Block Explorer**: Blockstream testnet explorer for verification
- **Browsers**: Chrome, Edge (Chromium-based)

### Critical Quality Areas
1. **Security**: User funds must be protected
2. **Reliability**: Operations must work correctly 100% of the time
3. **Usability**: Clear, intuitive user experience
4. **Performance**: Fast, responsive interactions
5. **Data Integrity**: No data loss or corruption

### Testing Constraints
- Fixed popup size: 600x400px
- Chrome extension environment
- Testnet transactions (real Bitcoin operations)
- Service worker can terminate unexpectedly

### Reference
See ARCHITECTURE.md for complete technical details, user flows, and feature specifications.

### 2. Functional Testing

**Core Feature Test Areas**

**A. Wallet Setup**
- Create new wallet (all address types)
- Import wallet from seed phrase
- Import wallet from private key (future)
- Password creation and validation
- Seed phrase display and security warnings
- Seed phrase confirmation
- Error handling (weak password, invalid seed, etc.)

**B. Wallet Unlock/Lock**
- Unlock with correct password
- Unlock with incorrect password
- Auto-lock after inactivity (15 minutes)
- Manual lock
- Unlock after service worker termination
- State persistence across lock/unlock

**C. Account Management**
- Create new account
- Rename account
- Switch between accounts
- Account balance display
- Multiple accounts handling
- Default account naming

**D. Send Transaction**
- Enter valid recipient address
- Enter invalid recipient address
- Enter amount (BTC)
- Amount validation (positive, sufficient funds)
- Fee selection (slow/medium/fast)
- Transaction preview
- Password confirmation
- Transaction signing
- Transaction broadcasting
- Success confirmation
- Transaction ID display
- Error handling (insufficient funds, network error)

**E. Receive Bitcoin**
- Display current address
- QR code generation
- Copy address to clipboard
- Generate new address
- Address format verification

**F. Transaction History**
- Display sent transactions
- Display received transactions
- Transaction details (amount, fee, confirmations)
- Transaction status (pending/confirmed)
- Block explorer link
- Empty state (no transactions)

### 3. Testnet Validation

**Testnet Testing Workflow**
```
1. Setup
   - Load extension in Chrome
   - Create new testnet wallet
   - Note down first receiving address

2. Receive Bitcoin
   - Go to testnet faucet: https://testnet-faucet.mempool.co/
   - Enter wallet address
   - Request testnet BTC
   - Wait for transaction
   - Verify balance updates in wallet
   - Check transaction in history
   - Verify on block explorer

3. Send Bitcoin
   - Generate second wallet address (for testing)
   - Send small amount (e.g., 0.001 BTC)
   - Verify transaction preview
   - Confirm transaction
   - Wait for broadcast
   - Verify transaction ID
   - Check on block explorer
   - Wait for confirmation
   - Verify balance updates

4. Multiple Confirmations
   - Track transaction confirmations
   - Verify confirmation count increases
   - Verify transaction status updates
```

**Testnet Test Scenarios**
- Receive from faucet
- Send to another wallet
- Send to same wallet (different account)
- Send with different fee rates
- Send maximum amount (all balance minus fee)
- Multiple transactions in sequence
- Unconfirmed transaction handling

**Bug Severity Definitions**
- **Critical**: Funds at risk, data loss, security vulnerability, app crash
- **High**: Core feature broken, no workaround, major usability issue
- **Medium**: Feature partially broken, workaround exists, minor usability issue
- **Low**: Cosmetic issue, typo, minor UI glitch

**Bug Priority Guidelines**
- **P0**: Must fix before release (blocker)
- **P1**: Should fix before release (high priority)
- **P2**: Can fix in next release (medium priority)
- **P3**: Nice to fix eventually (low priority)
